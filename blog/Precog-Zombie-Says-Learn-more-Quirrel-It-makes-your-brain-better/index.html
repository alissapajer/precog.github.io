Solve statements are one of the most important features of Quirrel. In conjunction with Quirrelï¿½s set-based nature, solve statements banish the spectre of traditional control structures such as for statements and while loops. It is possible to evaluate a statement at all possible values of a parameter in Quirrel by using a solve statement. All possible values of a parameter in a solve statement are determined by the constraints.

For example, in the solve statement below, ‘zombies is constrained to values of data.zombies. So, for each zombie you will get an object that contains two fields: monsterId and infections. The monsterId field will be populated with values from data.zombies and the infections field will be the results of the count on data.infections for each zombie.

solve 'zombies
  { monsterId: 'zombies,
  infections: count(data.infections where data.zombies =
       'zombies)
}
